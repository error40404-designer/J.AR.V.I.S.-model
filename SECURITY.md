1. Purpose
This policy ensures that vulnerabilities in JARVIS are identified, reported, and resolved quickly, while protecting users from unsafe use and encouraging responsible disclosure.

2. Vulnerability Management
- Monitoring: All code changes are reviewed for security risks before merging. Automated dependency scanning (e.g., GitHub Dependabot, CodeQL) is enabled.
- Reporting: Vulnerabilities can be reported via GitHub’s Security Advisories or by opening a private issue.
- Response Time:
- Acknowledgement: Within 48 hours of receiving a report.
- Assessment: Within 7 days, severity is classified (Critical, High, Medium, Low).
- Resolution: Critical/High vulnerabilities patched within 14 days; Medium/Low within 30 days.
- Communication: Security advisories will be published for confirmed vulnerabilities, with clear mitigation steps.

3. Responsible Disclosure
- Researchers and users are encouraged to report vulnerabilities privately.
- Public disclosure should only occur after a patch is released.
- No legal action will be taken against good‑faith security researchers.

4. Secure Usage Guidelines
- Do not share personal or private data (passwords, financial info, medical info, etc.) with JARVIS.
- Avoid unsafe commands: JARVIS should not be used to execute harmful or destructive actions (e.g., deleting system files, bypassing security).
- Safe communication: Users must avoid unsafe or abusive language. JARVIS is designed to reject harmful prompts.
- Privacy first: JARVIS anonymizes prompts in its database and sanitizes sensitive information before storage.

5. Continuous Improvement
- Regular audits: Codebase reviewed quarterly for security risks.
- Dependency updates: Automated tools ensure libraries are patched.
- User feedback loop: Issues raised by users are tracked and resolved transparently.

6. Enforcement
- Any misuse of JARVIS (e.g., attempts to exploit vulnerabilities, unsafe prompts) may result in restricted access.
- Contributors must follow secure coding practices and respect privacy guidelines.
